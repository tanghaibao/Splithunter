TcellExTRECT cov_example — upstream-style exon report

This snapshot reflects Splithunter's corrected TcellExTRECT implementation. For WES-style runs with exon intervals, the estimator now follows the original pipeline semantics: exon-selected coverage, per-exon running medians, low-depth exon removal, smoothed focal log-ratio estimation, and QC reporting.

Why this changed. The older Splithunter port treated exon BEDs as regions to exclude from a whole-window focal-vs-baseline median. Upstream TcellExTRECT does the opposite: exons are the signal substrate. The focal TRA window is intronic, so under the corrected pipeline this fixture has 0 focal exon positions and the score comes from the smoothed exon log-ratio trend across the locus rather than an intronic median.

Interpretation

The fixture still shows a real T-cell signal, but the corrected port no longer reports the old 50% number because that value came from the simplified whole-window shortcut. The upstream-style path uses only exon-supported signal, drops one failed exon, and estimates the focal dip from the smoothed exon profile. That yields a materially lower but internally consistent estimate with a finite QC score.

Reproduce

python -c "
import csv, gzip, os.path as op
from splithunter import tcell, loci

fix = 'tests/fixtures'
pos, dep = [], []
for row in csv.DictReader(gzip.open(op.join(fix, 'cov_example.csv.gz'), 'rt')):
    pos.append(int(row['pos']))
    dep.append(int(row['reads']))

segs = {r['segName']: (int(r['start']), int(r['end']))
        for r in csv.DictReader(open(op.join(fix, 'tcra_seg_hg19.csv')))}
seg = loci.LocusSegments('TRA', 'chr14',
                         segs['all'], segs['focal'],
                         segs['local1'], segs['local2'])
exons = loci.load_exons_bed(op.join(fix, 'tcra_exons_hg19.bed'), 'chr14')

print(tcell.estimate_from_coverage(pos, dep, seg, target_intervals=exons, min_cov=0))
"

Use --no-default-targets only if you explicitly want the older whole-window approximation.
